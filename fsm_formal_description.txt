ФОРМАЛЬНОЕ ОПИСАНИЕ СИМВОЛЬНОГО АВТОМАТА ПЕРЕДАТЧИКА
==================================================

1. МНОЖЕСТВО СОСТОЯНИЙ:
   S = {S0, S1, S2, S3, S4, S5}
   где:
   S0 = IDLE        (000) - состояние покоя
   S1 = START_BIT   (001) - передача стартового бита
   S2 = DATA_BITS   (010) - передача битов данных
   S3 = PARITY_BIT  (011) - передача бита четности
   S4 = STOP_BIT    (100) - передача стопового бита
   S5 = NEXT_CELL   (101) - выбор следующей ячейки

2. ВХОДНОЙ АЛФАВИТ:
   I = {i0, i1, i2, i3, i4, i5, i6}
   где:
   i0: action = 0 (бездействие)
   i1: action = 1 (запись в матрицу)
   i2: action = 2 (передача одной ячейки)
   i3: action = 3 (передача строки)
   i4: action = 4 (передача столбца)
   i5: action = 5 (передача всей матрицы)
   i6: div_counter = DIV-1 (достигнут счетчик делителя)
   i7: bit_counter = W-1 (переданы все биты)
   i8: cells_sent = cells_to_send (переданы все ячейки)

3. ВЫХОДНОЙ АЛФАВИТ:
   O = {o0, o1, o2, o3, o4, o5, o6}
   где:
   o0: tx = 1 (линия в состоянии покоя/стоповый бит)
   o1: tx = 0 (стартовый бит)
   o2: tx = данные (биты данных)
   o3: tx = бит четности
   o4: busy = 0 (не занят)
   o5: busy = 1 (занят)
   o6: обновить индексы ячеек

4. ФУНКЦИЯ ПЕРЕХОДОВ δ: S × I → S:

   δ(S0, i1) = S0    // action=1, остаемся в IDLE
   δ(S0, i2..i5) = S1 // action=2..5, переход в START_BIT
   δ(S0, i0) = S0    // action=0, остаемся в IDLE

   δ(S1, ¬i6) = S1   // div_counter < DIV-1, остаемся в START_BIT
   δ(S1, i6) = S2    // div_counter = DIV-1, переход в DATA_BITS

   δ(S2, ¬i7) = S2   // bit_counter < W-1, остаемся в DATA_BITS
   δ(S2, i7 ∧ PAR=0) = S4 // все биты переданы, PAR=0, переход в STOP_BIT
   δ(S2, i7 ∧ PAR≠0) = S3 // все биты переданы, PAR≠0, переход в PARITY_BIT

   δ(S3, ¬i6) = S3   // div_counter < DIV-1, остаемся в PARITY_BIT
   δ(S3, i6) = S4    // div_counter = DIV-1, переход в STOP_BIT

   δ(S4, ¬i6) = S4   // div_counter < DIV-1, остаемся в STOP_BIT
   δ(S4, i6 ∧ ¬i8) = S5 // все биты, но не все ячейки, переход в NEXT_CELL
   δ(S4, i6 ∧ i8) = S0 // все биты и все ячейки, переход в IDLE

   δ(S5, *) = S1     // всегда переходим в START_BIT

5. ФУНКЦИЯ ВЫХОДОВ λ: S → O:

   λ(S0) = {o0, o4}        // tx=1, busy=0
   λ(S1) = {o1, o5}        // tx=0, busy=1
   λ(S2) = {o2, o5}        // tx=данные, busy=1
   λ(S3) = {o3, o5}        // tx=четность, busy=1
   λ(S4) = {o0, o5}        // tx=1, busy=1
   λ(S5) = {o5, o6}        // busy=1, обновить индексы

6. НАЧАЛЬНОЕ СОСТОЯНИЕ:
   s0 = S0 (IDLE)

7. ГРАФ АВТОМАТА:
   Вершины: состояния S0..S5
   Дуги: переходы согласно функции δ
   Метки дуг: условия перехода
   Метки вершин: выходные сигналы согласно функции λ

8. СВОЙСТВА АВТОМАТА:
   - Детерминированный (для каждого состояния и входа определено следующее состояние)
   - Полностью определенный (для всех пар состояние-вход определен переход)
   - Конечный (конечное число состояний)
   - Автомат Мили (выходы зависят от состояния)
   - Синхронный (переходы по тактовому сигналу)

9. ПРИМЕР РАБОТЫ АВТОМАТА для action=2 (передача одной ячейки):
   S0 → (i2) → S1 → (i6) → S2 → (i7) → S4 → (i6) → S0
   Цикл: IDLE → START_BIT → DATA_BITS → STOP_BIT → IDLE

10. ПРИМЕР РАБОТЫ АВТОМАТА для action=3 (передача строки, 4 ячейки):
    S0 → S1 → S2 → S3 → S4 → S5 → S1 → S2 → S3 → S4 → S5 → ... (4 раза)
    Последний цикл: S4 → (i6 ∧ i8) → S0
